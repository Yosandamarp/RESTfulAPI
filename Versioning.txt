Saat membuat RESTful API,usahakan tidak pernah merusak kompatibilitas 
tiap kita melakukan upgrade
Versioning hanya diperlukan ketika kita memang perlu merusak kompatibilitas
RESTful API kita
Disarankan sebisa mungkin tidak pernah melakukan ini
Versioning adalah memberi tahu bahwa kita memiliki banyak versi terhadap 
aplikasi RESTful API kita 
Versioning pada URL
http://api.example.com/v1/products
http://api.example.com/v2/products
http://api.example.com/2019/products
http://api.example.com/2020/products
